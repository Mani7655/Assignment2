Semantic Elements

When building web page,we use a combination of non-semantic HTML and semantic HTML.

The word semantic means "relating to meaning" ,so semantic element privide information about the content between the opening and closing tag.

By using semantic HTML we select html element based on their meaning, not on how they are presented.Element such as "div"and "span"are not semantic element since they provide no context as to what information inside of those tags.

Example:

For example, instead of using a "div" element to contain our header information, we could use a "header"tag.

Why use semantic elements?

  • Accessibility: Semantic HTML makes web page accessible for mobiledevices and for people with disabilities as well
  • SEO: it improves the website SEO.With better SEO, search engine are better able to identify the content of your website and weight your content.
  • Easy to Understand: Semantic HTML also make the website source code easier to read for other web developers.

Many websites contain HTML code like:<div id="header" > , <div id="nav" > , <div id="main" > <div id="footer" > , to indicate header, navigation, main content and footer.
